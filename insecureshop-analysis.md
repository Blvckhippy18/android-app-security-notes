# InsecureShop Security Analysis

## Objective
To analyze intentionally vulnerable Android applications and identify insecure coding practices.

## Findings
- Insecure data storage
- Hardcoded secrets
- Weak SSL validation
- Exported activities

## Tools Used
- MobSF
- Android Emulator
- ADB

## Lessons Learned
Mobile applications require secure storage mechanisms and proper validation controls to protect user data.
